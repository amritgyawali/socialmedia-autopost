# Research corrections to the supplied build plan

Checked against first-party documentation on **2026-07-16**. Platform terms, quotas, prices, API versions, and review rules can change; re-open the linked source before spending money or submitting an app for review.

## Architecture and implementation corrections

- This repository implements the **custom Spring Boot engine path**. It does not bundle or deploy Postiz. Do not combine Postiz's separate official stack with this database unless you deliberately design and test that migration.
- The browser never receives `COCKPIT_JWT_SECRET`, platform client secrets, R2 credentials, the vault key, or database credentials. Next.js route handlers are the server-side UI-to-engine adapter.
- `CORS_ALLOWED_ORIGINS` limits browser behavior; it does not stop `curl`, bots, or server-to-server traffic. The JWT signature, issuer/audience checks, short expiration, TLS, and unexposed internal ports are the controls that protect the engine.
- Media is public because publishing providers need to fetch it. Do not upload confidential assets to the public media bucket. Random object keys reduce accidental guessing but do not make a public URL private.
- The current LinkedIn and X adapters buffer downloaded media in Java heap. Production therefore aligns upload and publisher limits at 25 MiB. Larger-video support requires a tested streaming implementation; a 1 GB presign limit would otherwise advertise files this v1 engine cannot safely publish.
- Private-backup R2 credentials and `RESTIC_PASSWORD` are host-only. Compose explicitly allowlists the values passed to the engine instead of injecting the entire operator `.env` file.
- A successful `pg_dump` command alone is not a backup strategy. The included flow encrypts dumps with restic, sends them off-host to a private R2 bucket, retains multiple generations, and includes a verification script that restores and parses an archive.
- “Approval not required for my own account” is never a universal platform promise. The exact access level depends on the product, scopes, account type, asset ownership, app roles, and current platform dashboard. Test each target account before depending on it operationally.
